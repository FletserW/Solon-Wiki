O que é FTP, como acessar e configurar no seu servidor

O FTP, ou Protocolo de Transferência de Arquivos, desempenha um papel fundamental no funcionamento adequado de um site WordPress. Ele permite que os usuários realizem a transferência de arquivos e resolvam problemas que possam surgir ao operar o site.

Entender como ocorre a transferência de dados entre o dispositivo do usuário e o serviço de hospedagem, bem como a estrutura dos diretórios do WordPress, é essencial para garantir o bom funcionamento do projeto. Assim como um equipamento precisa dos comandos corretos para operar, um site requer uma estrutura apropriada para alcançar um desempenho satisfatório.

O FTP possibilita o acesso à estrutura de arquivos do site, o que permite aos usuários resolver problemas decorrentes de alterações no código do template, incompatibilidade de extensões ou instalação inadequada de plugins e outros elementos.

Neste artigo, abordaremos o funcionamento do FTP, explicando como ele realmente funciona, os problemas que podem ser solucionados por meio dele e outros aspectos relevantes para utilização efetiva dessa ferramenta no contexto do WordPress.

O que é FTP?

O FTP (Protocolo de Transferência de Arquivos) estabelece uma conexão entre o dispositivo do usuário e um servidor, geralmente pertencente ao serviço de hospedagem escolhido. Essa conexão permite a troca de dados entre os sistemas quando estão conectados à internet.

O FTP foi desenvolvido na década de 1970 como parte do avanço da tecnologia computacional e das primeiras redes e conexões. Criado por Abhay Bhushan, que era estudante de Engenharia Elétrica no Massachussets Institute of Technology na época, o objetivo principal do FTP era criar um protocolo de transferência mais seguro, capaz de prevenir tentativas de intrusão e roubo de informações.

Com o passar do tempo e as melhorias no código e capacidade de armazenamento dos equipamentos, o FTP se consolidou como um dos métodos mais eficientes de compartilhamento de dados na internet.

O fluxo de transferência de dados ocorre por meio de um programa que conecta dois agentes: o cliente, que solicita a conexão e acesso aos dados hospedados, e o servidor, uma infraestrutura mais robusta composta por um ou mais equipamentos responsáveis por operações e armazenamento de informações. O servidor recebe a solicitação do cliente e concede autorização com base na política de acesso adotada, geralmente exigindo um login e senha.

Como instalar o FTP?

Ao acessar o FTP, você precisa utilizar uma aplicação específica que requer um login e uma senha para autorização. Além disso, é importante especificar a porta de conexão correta.

Existem diferentes opções de aplicativos de FTP disponíveis, e você pode escolher aquele que melhor atende às suas necessidades. Alguns exemplos populares incluem:

  • FileZilla;
  • Free FTP;
  • ProFTPd;
  • SmartFTP;
  • Pure-FTPd;
  • VsFTPd;
  • Titan FTP Server;
  • Cyberduck;
  • Kasablanca;
  • WinSCP

Você pode obter o programa de FTP adequado para você, fazer o download e instalá-lo em seu computador. Esses aplicativos são especialmente úteis para operações de arquivo mais complexas e robustas.

Se o seu serviço de hospedagem oferecer um aplicativo da web para acesso ao FTP, você poderá encontrá-lo na interface do usuário ou no painel de controle. No entanto, se você preferir utilizar um programa de terceiros, recomendamos escolher uma das opções mencionadas anteriormente.

Dessa forma, você estará pronto para aproveitar todos os recursos e benefícios do FTP ao gerenciar seus arquivos e realizar operações de transferência de dados de forma eficiente.dade, faça o download e instale o cliente de FTP no seu computador.

Como configurar o FTP?

Se a sua hospedagem WordPress for gerenciada, como a do Stage, é recomendado consultar o suporte técnico para obter orientações sobre como instalar, configurar e acessar o FTP. No entanto, se você não possui uma hospedagem gerenciada, siga os seguintes passos:

  • Inicie o programa de FTP previamente instalado em seu computador.
  • Digite o endereço quando solicitado. Essa informação varia de acordo com o provedor de hospedagem, mas geralmente os dados estão disponíveis no painel de controle do servidor. No cPanel, por exemplo, você pode encontrar essas informações em "Contas FTP". O endereço costuma ser o mesmo que o nome de domínio com o prefixo "ftp" (ftp.seusite.com).
  • Insira o seu nome de usuário e senha, que são os mesmos definidos ao criar a sua conta ou utilizados para acessar o cPanel. Digite a porta de acesso FTP. O valor padrão é 21 para FTP e 22 para SFTP.
  • Clique no botão "Connect" ou "Quickconnect" para estabelecer a conexão com a sua conta de hospedagem.
Caso o seu domínio não esteja apontando para o seu host, você pode tentar utilizar o IP do servidor, que é especificado nas configurações. Vamos supor que você tenha escolhido o FileZilla como o programa de FTP para realizar essas etapas. Como usar o FTP?

Para copiar algum arquivo, basta localizá-lo em seu computador na parte esquerda do programa e jogá-lo para a parte direita em seu servidor.

Entretanto, antes de usar o FTP, você deve saber como é feita a estruturação dos arquivos e diretórios no WordPress. Isso é essencial para que o CMS funcione adequadamente, uma vez que a falta de um único arquivo pode derrubar o seu site e causar vários problemas sobre os quais falaremos a seguir.

Além disso, conhecer e saber como acessar os arquivos e diretórios no WordPress pode ajudar você a corrigir vários problemas que podem surgir. Por exemplo, um plugin instalado inadequadamente que bloqueou o acesso ao dashboard de administração do site — a única forma de excluí-lo é por meio do acesso ao FTP.

Pastas

A estrutura do diretório é formado pelas seguintes pastas.

wp-admin

A pasta wp-content tem dois subdiretórios: plugins e temas, que armazenam dados relacionados a essas extensões no WordPress. A cada instalação, uma nova subpasta é criada com o nome do elemento adicionado.

Arquivos

Além das pastas, o diretório do WordPress, cujo acesso acontece por meio do FTP, é formado por arquivos de configuração, como:

  • htaccess — usado para gerenciar redirecionamentos e links permanentes;
  • wp-config.php — que ajuda o WP a se conectar com o banco de dados e delimita algumas configurações para os sites;
  • o index.php — arquivo principal, por meio do qual o WP carrega os posts que serão exibidos e usados como substituição sempre que faltar um arquivo base do CMS.

Além desses, o style.css — um arquivo CSS declarativo e necessário para a criação de temas no WordPress — controla a apresentação das páginas. Veja outros arquivos importantes a seguir:

  • license.txt
  • readme.html
  • wp-activate.php
  • wp-blog-header.php
  • wp-comments-post.php
  • wp-config-sample.php
  • wp-cron.php
  • wp-links-opml.php
  • wp-load.php
  • wp-login.php
  • wp-mail.php
  • wp-settings.php
  • wp-signup.php
  • wp-trackback.php
  • xmlrpc.php
  • robots.txt

É recomendado que o usuário mantenha a estrutura padrão, pois pequenas modificações podem inviabilizar todo o funcionamento do site. O arquivo zip do WP para download contém todos esses componentes e o usuário precisa apenas extraí-los e enviá-los ao seu serviço de hospedagem por meio do FTP.

Caso você acesse o FTP sem precisar baixar um programa, digite “ftp://número do ip” no navegador para que um prompt de comando solicite o login e senha de acesso.

Quais os problemas no WordPress posso resolver pelo FTP?

Ter acesso via FTP permite que o usuário corrija vários erros e problemas comuns no WordPress. Veja alguns deles a seguir.

Erros de conexão de banco de dados no WordPress

Se ocorrer um erro ao estabelecer uma conexão com o banco de dados no WordPress, a primeira ação que você deve realizar é entrar no painel de administração. Caso apareça o aviso “erro de banco de dados indisponível”, você precisa reparar essa conexão por meio do acesso FTP:

  1. Acesse a pasta raiz do seu WordPress e edite o arquivo wp-config.php.
  2. Adicione a seguinte linha ao final do arquivo: define('WP_ALLOW_REPAIR', true);
  3. Depois de salvar e recarregar o arquivo wp-config.php, visite a seguinte URL em seu navegador: www.seudominio.com/wp-admin/maint/repair.php. Isso abrirá um utilitário de manutenção do WordPress que permite reparar seus bancos de dados. Clique no botão para iniciar o processo.
  4. Depois de terminar o processo, edite o arquivo wp-config.php novamente para remover a linha que você adicionou.

Erro de memória esgotada do WordPress

Caso o seu site ou blog do WordPress mostre um erro de memória esgotada, isso também pode ser corrigido com acesso ao FTP. A correção permite adicionar mais elementos de personalização, como plugins. A capacidade de memória padrão para WordPress é de 64 MB, o que geralmente é suficiente para a maioria dos casos. No entanto, pode haver situações em que o usuário necessite de mais.

Todas as funções de um site em WordPress demandam memória, assim como um computador. Para aumentar a memória do seu WP, siga estes passos:

  1. Acesse o diretório FTP e abra a pasta raiz do WordPress.
  2. Procure pelo arquivo wp-config.php.
  3. Clique no arquivo e digite na última linha: define('WP_MEMORY_LIMIT', '256M');

    4. Isso definirá o limite de memória alocada para 256 MB, o que deve ser mais do que suficiente para que o erro de esgotamento de memória seja resolvido.

“Tela Branca da Morte”

Imagine o seguinte: você tenta acessar seu site e tudo o que obtém é uma tela completamente branca. Isso geralmente acontece quando plugins ou templates com erros estão ativos no WordPress. Entretanto, esse problema é crítico, porque também impede o acesso ao painel de administração do WordPress.

Nessas situações, a única solução é conectar-se ao arquivo WordPress e à estrutura de diretório por meio do acesso FTP. Se você já supõe que o erro está em um plugin, o caminho mais fácil é a desativação em bloco, de todos de uma vez. Dessa forma, se o erro não ocorrer mais, você precisará renomear cada arquivo individualmente para testar. Então vamos ao passo a passo:

  1. Acesse o FTP.
  2. Encontre o diretório raiz onde instalou e faz uso do WordPress.
  3. Localize a pasta wp-content/plugins.
  4. Renomeie a pasta inteira (escolha qualquer outro nome, mas não se esqueça que você precisará refazer esse passo depois, então lembre-se do nome escolhido).

Pronto, agora todos os plugins estão desativados. Acesse novamente a página e veja se o problema foi resolvido. Se sim, você precisará renomear cada arquivo individualmente, por meio dos passos a seguir:

  1. Acesse o FTP.
  2. Encontre o diretório raiz onde instalou e faz uso do WordPress.
  3. Localize a pasta wp-content/plugins.
  4. Renomeie a pasta inteira novamente para plugins.
  5. Restrinja plugin por plugin da mesma forma, por meio da alteração do nome da pasta.
  6. Sempre que fizer isso, teste o site.

O plugin que causou o erro deve ser deletado. Se o problema não resolveu com essa ação, provavelmente diz respeito aos temas. Vá para os diretórios wp-content/themes e renomeie-os. Verifique se você consegue abrir o painel de administração do WordPress agora.

Erro de upload de imagem no WordPress

Caso você tenha problemas para realizar o upload de imagens em seu site ou blog do WordPress, uma possível solução envolve o acesso ao FTP. Isso ocorre porque uma das causas

  1. Acesse a pasta "wp-content" do WordPress no diretório raiz.
  2. Procure pela pasta "uploads".
  3. Abra as configurações de permissões dessa pasta.
  4. Redefina as permissões para a pasta "uploads" e seus subdiretórios para 744.
  5. Ative a opção "recurse" para aplicar a alteração a todas as pastas simultaneamente.
  6. Defina as permissões para os arquivos na pasta "uploads" para 644.
  7. Ative a opção "recurse" em subdiretórios.

Depois de aplicar todas as alterações, volte ao painel de administração e tente enviar uma imagem para uma postagem novamente.

Como você pode ver, o FTP também é uma opção ao painel administrativo em caso de erros, principalmente se esse problema inviabiliza o acesso à interface principal.
